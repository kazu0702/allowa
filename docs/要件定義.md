# INCE 要件定義

## 1. 概要

INCEは、親がこどものテスト・成績・学習成果に応じてポイントを付与し、こどもがそのポイントをお小遣いとして換金申請できるスマホ向けWebアプリである。

ポイントは `1pt = 1円` として扱う。

未ログイン時のトップページはLPとし、ログイン後は利用者種別に応じて保護者ホームまたはこどもホームへ遷移する。

## 2. 利用者

### 保護者

- こどもを追加・管理する。
- こどもごとの科目・ポイントルール・換金単位を設定する。
- こどもからの申請を確認し、承認・修正・差し戻し・却下する。
- こどもの代わりに代理申請できる。
- 換金申請を確認し、お小遣い支給後に完了処理を行う。

### こども

- 招待リンクからアカウント登録する。
- メールアドレスは持たない前提とする。
- ログインIDとパスワードでログインする。
- テスト・成績・その他の学習成果を申請する。
- ポイント残高を確認し、換金申請する。

## 3. 料金・契約

- 14日間無料トライアルを提供する。
- 月払いは月額500円とする。
- 年払いは年額5,000円とする。
- こどもは最大3人まで登録可能とする。
- トライアル終了後は自動継続課金とする。
- 支払い失敗時は7日間の猶予期間を設ける。
- トライアル中に退会した場合、トライアル満了日までは利用可能とする。
- トライアル満了日の翌日以降はログイン不可とする。

### ログイン可否

| 契約状態 | ログイン |
|---|---|
| 14日間トライアル中 | 可能 |
| 有料契約中 | 可能 |
| トライアル中に退会済み・満了前 | 可能 |
| トライアル中に退会済み・満了後 | 不可 |
| 支払い失敗・7日間の猶予期間中 | 可能 |
| 支払い失敗・猶予期間終了後 | 不可 |
| 退会済み | 不可 |

## 5. 申請分類

申請時に以下のいずれかを選択する。

- テスト
- 成績
- その他

保護者は申請確認時に、分類・科目・点数・評価・ポイントを修正できる。

## 6. テスト申請

### 入力項目

- 科目
- 満点種別: 100点満点 / 50点満点
- 点数
- 写真

### ルール

- 100点満点と50点満点は別々にポイント設定できる。
- ポイントルールはこどもごとに設定する。
- 親は段階制または計算制を選択できる。
- 初期値はイージー・ノーマル・ハードから選択して自動反映できる。
- 自動反映後、親が自由に編集できる。

### 段階制

点数帯ごとに付与ポイントを設定する。

例:

| 点数条件 | 付与ポイント |
|---:|---:|
| 100点 | 1,000pt |
| 90点以上 | 700pt |
| 80点以上 | 300pt |

### 計算制

点数に応じてポイントを計算する。

設定例:

- 1点あたりのポイント
- 最低点
- 上限ポイント

## 7. 成績申請

### 入力項目

- 科目
- 評価種別: 5段階評価 / A・B・C評価
- 評価
- 写真

### ルール

- 5段階評価とA/B/C評価は別々にポイント設定できる。
- ポイントルールはこどもごとに設定する。
- 初期値はイージー・ノーマル・ハードから選択して自動反映できる。
- 自動反映後、親が自由に編集できる。

## 8. その他申請

分類が「その他」の場合は、固定ルールではなく都度判断とする。

### 入力項目

- 内容
- 科目
- 写真
- おまかせ、または希望ポイント

### ルール

- こどもは「おまかせ」または「希望ポイント数を入力」を選択できる。
- 保護者は承認時にポイント数を決定または修正できる。
- 承認時に確定ポイントを付与する。

## 9. 科目

- 科目は保護者が自由に追加できる。
- 科目はこどもごとに管理する。
- 科目名は編集・削除できる。
- 削除済み科目が過去履歴に影響しないよう、申請履歴側には当時の科目名を保存する。

## 10. 代理申請

こどもがスマホを持っていない場合、保護者がこどもの代わりに申請できる。

### 流れ

1. 保護者がこどもを選択する。
2. 代理申請を作成する。
3. 分類・科目・点数または評価・写真を入力する。
4. 確認画面を表示する。
5. 保護者が確定する。
6. 承認済みとして保存する。
7. ポイントを即時付与する。
8. 写真は承認済み写真として60日間保存する。

## 11. ポイント

- `1pt = 1円` とする。
- 申請承認時にポイントを付与する。
- 換金完了時にポイントを消費する。
- 承認取り消し時はポイント付与も取り消す。
- ただし、対象ポイントが換金済みの場合は承認取り消し不可とする。
- 現在ポイントだけでなく、ポイント増減履歴を必ず保存する。

### 月次ボーナス

月次ボーナスは、こどもの申請なしで保護者が任意に付与できる追加ポイントとする。

### 家庭内ルール候補

月次ボーナス画面では、家庭内ルールとして使いやすいボーナス候補を表示する。

- 表示される追加ポイント候補は目安であり、自動付与はしない。
- 保護者は月ごとに「付与する / 付与しない」を選択できる。
- 付与する場合、保護者が最終的な付与ポイント数を確認・調整できる。
- 付与確定時にポイント履歴へ記録する。
- 候補名、対象月、基準ポイント、目安割合、計算結果を保存する。
- こども向けには家庭内のボーナスとして表示する。

### 親独自ボーナス

保護者は、候補とは別に、家庭独自の月次ボーナスを作成できる。

例:

- 誕生月ボーナス
- 皆勤ボーナス
- 読書ボーナス
- お手伝いボーナス

ルール:

- こどもの申請なしで付与できる。
- 保護者がボーナス名、対象こども、付与ポイント、対象月、メモを設定できる。
- 毎月自動候補として表示するか、単発付与にするかを選択できる。
- 最終的な付与は保護者の確認操作を必須とする。
- 付与済みの月次ボーナスは、月別付与履歴とポイント履歴に表示する。
- 誤付与に備え、未換金分については取り消しできる。

## 12. 換金申請

保護者がこどもごとに換金単位を設定する。

- 100pt単位
- 1,000pt単位
- 10,000pt単位

複数選択は不可とする。

こどもは保有ポイント内で、設定単位に従って換金申請できる。

換金完了時には、保護者がお小遣いを支給したものとしてポイントを消費し、換金履歴に保存する。

## 13. 写真保存

### 保存期間

| 状態 | 写真の扱い |
|---|---|
| 申請中 | 保存する |
| 承認済み | 60日間保存し、その後自動削除 |
| 差し戻し | 30日間保存し、その後自動削除 |
| 却下 | 30日間保存し、その後自動削除 |
| 期限経過 | 自動削除 |

### アップロード仕様

- 1申請につき写真1〜3枚とする。
- 1枚あたり最大10MBを目安とする。
- 1申請あたり合計20MB程度までを目安とする。
- JPEG、PNG、HEICに対応する。
- アップロード時または保存時に自動圧縮する。
- 保存時は長辺1600px程度にリサイズする。

## 14. 重複申請防止

重複申請はシステムで自動ブロックしない。

- 親が写真を見て判断する。
- 承認済み写真を60日間参照できる。
- 申請確認時に直近60日以内の承認済み写真を見られる。
- 「似た申請がないか確認してください」という注意表示は出す。

この方針により、保護者が写真を確認せずに適当に承認することを防ぐ。

## 15. 承認取り消し

承認後に誤りが見つかった場合、保護者は承認を取り消せる。

### 条件

- 付与されたポイントがまだ換金されていないこと。
- 換金済みポイントを含む場合は取り消し不可とする。

### 処理

- 申請ステータスを承認取り消しに変更する。
- 付与済みポイントを減算する。
- ポイント履歴に取り消し記録を残す。
- 月別履歴にもマイナス履歴として表示する。

## 16. 通知

MVPではアプリ内通知を基本とする。

### 保護者向け通知

- 新しい申請があります
- 換金申請があります
- 差し戻し中の申請があります
- 支払い方法を確認してください

### こども向け通知

- 申請が承認されました
- 申請が差し戻されました
- 申請が却下されました
- 換金が完了しました

こどもがスマホを持たない場合は、保護者画面で通知・履歴を確認する。

## 17. UI方針

- ほぼスマホ専用のUIとする。
- PC対応は最低限とする。
- 下部タブナビゲーションを採用する。
- 1画面1目的を基本にする。
- ボタンは大きめにする。
- 入力はステップ式にする。
- 写真撮影・写真確認を中心にした設計にする。
- オレンジ系の明るい配色にする。
- シンプルで実用重視にする。
- 横スクロールは使わない。
- 履歴や一覧はカード表示にする。
- 重要操作は確認画面を挟む。
- 親子で同じスマホを使う可能性があるため、アカウント切り替えを分かりやすくする。

## 18. キャラクター画像

キャラクター画像は使用しない。

### 使用方針

- LP、完了画面、空状態、通知、申請・承認フローの補助演出にはキャラクター画像を使わない。
- 実用寄りのUI、ロゴ、アイコン、余白、色で親しみやすさを表現する。
- キャラクター素材ディレクトリは保持しない。

### 主な使用箇所

- LPのファーストビュー
- 申請完了画面
- 承認完了画面
- ポイント獲得画面
- 換金申請完了画面
- 履歴がない状態
- 入力エラーや差し戻し時の案内
