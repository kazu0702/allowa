1. 事業概要

INCEは、子どもの学習成果を保護者が確認し、ポイントとして付与し、おこづかい申請までつなげる親子向け学習モチベーションアプリである。

  1. 子どもがテスト・成績・学習成果を申請する。
  2. 保護者が内容を確認し、承認・差し戻し・却下する。
  3. 承認時にポイントを付与する。
  4. 子どもがポイントを貯め、おこづかい申請する。
  5. 保護者が支給完了する。

2. 市場前提

2024年の国民生活基礎調査では、児童のいる世帯は907.4万世帯、全世帯の16.6%とされている。INCEはこのうち、学習習慣づけ・親子確認・おこづかい管理に関心がある家庭を初期ターゲットとする。

初期の獲得対象は、児童のいる世帯のうち0.01%から0.15%程度を現実的なレンジとして設定する。

8. 年度別戦略

1年目: MVP検証と初期課金化

  • 30〜100世帯の実利用を作る。
  • 年末1,000課金世帯を目指す。
  • LP、SNS、子育てコミュニティで検証する。
  • 保護者インタビューを継続する。

2年目: β版拡大と継続率改善

  • 課金世帯数4,000世帯を目指す。
  • 本番DB・認証・決済を導入する。
  • SEO、SNS、紹介施策を始める。
  • サポート体制を整える。

3年目: 事業化と利益化

  • 年末12,000課金世帯を目指す。
  • 広告運用、口コミ、紹介プログラムを本格化する。
  • 年払い比率を高め、解約率を3.0%以下に抑える。
  • 月次振り返り、家庭内ルール管理、学習習慣化テンプレートを強化する。

10. 主要リスクと対策

リスク対策
課金転換率が低い無料期間中に価値を感じる導線を強化する。
継続率が低い通知、履歴、月次振り返りを強化する。
子どもの心理的負担点数だけでなく努力・その他申請を重視する。
保護者の入力負担デフォルト設定とワンタップ承認を整える。
CAC高騰SEO、紹介、コミュニティ経由を重視する。
個人情報・写真管理保存期限、削除、セキュリティを本番前に整備する。

11. 次に検証すべきこと

  1. 10〜30世帯にMVPを触ってもらう。
  2. 子どもが自分で申請したくなるかを見る。
  3. 保護者が承認作業を面倒に感じないかを見る。
  4. 月500円を払いたいと言う家庭がどの程度あるか確認する。
  5. 継続利用されるポイントルールの形を見つける。

最初の目標は、30世帯中10世帯が「課金してもよい」と答える状態を作ること。
