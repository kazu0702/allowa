# 本番セキュリティメモ

## 現状の位置づけ

現在のログイン状態保持は、MVP・動作確認としては問題ない。
ただし、本番課金ユーザーに提供する前には、認証・パスワード管理・セッション管理を本番仕様に作り替える必要がある。

## 本番前に対応したいこと

- 親アカウントは Supabase Auth などの正式な認証にする。
- パスワードは平文保存しない。
- こどもログインもサーバー側で検証する。
- 重要操作には親パスワードの再入力を入れる。
- 一定期間未使用なら再ログインを求める。
- ログアウトボタンは分かりやすく配置する。
- XSS対策として、入力値の扱い、外部スクリプト、HTML挿入を慎重にする。

## 注意点

`localStorage` によるログイン状態保持は、端末内に情報が残る。
共有端末や紛失端末では、ログアウトしない限り再度アプリを開ける可能性がある。

また、XSSが起きた場合に `localStorage` の情報を読まれるリスクがあるため、本番では認証基盤とサーバー側のセッション管理を前提に設計する。
