## 2. UI改善で見るべき画面

### 優先度 高

- `/parent`
  - 保護者ホーム
  - 月次ボーナス、申請、おこづかい、こども管理への導線整理

- `/parent/monthly-bonus`
  - 今回追加した新機能
  - 家庭内ルールのボーナス候補表示
  - 付与する / 付与しない
  - 親独自ボーナス
  - 履歴表示

- `/child`
  - こどもホーム
  - 現在ポイント、申請、履歴、おこづかい申請への導線

### 優先度 中

- `/child/apply`
  - こどもの申請作成

- `/child/history`
  - 申請履歴
  - 写真サムネイル
  - 修正、キャンセル、再申請、削除

- `/parent/applications`
  - 保護者の申請確認一覧

- `/parent/redemptions`
  - おこづかい申請確認

### 優先度 低

- `/parent/settings`
- `/parent/billing`
- `/parent/demo-guide`
- `/admin`

## 3. 現在の実装メモ

- アプリは静的HTML/CSS/JSで構成されている。
- `app/app.js` が大きな単一ファイルになっている。
- `app/monthly-bonus.js` は、公開反映を早くするために後付けで追加したファイル。
- Vercelのビルドでは `scripts/build.mjs` が `app` 配下を `dist` にコピーしている。
- UI改善時は基本的に `app/styles.css` と画面HTML生成部分を触る。
- Vercel反映にはGitHub側への反映が必要。

## 4. デザイン方針

- スマホファースト
- 1画面1目的
- ボタンは大きめ
- 親が迷わず使える実用寄り
- こどもには達成感が伝わる
- 空状態、完了、導入部分でもキャラクター画像は使わない
- カードの使いすぎで画面が重くならないようにする
- 金融アプリっぽくしすぎない
- 月次ボーナスは「家庭内の追加ポイント」として見せる

## 5. 避けたい表現

以下は使わない。

- 投資
- 運用
- 利回り
- 年利
- 月利
- 配当
- 元本
- 元本保証
- 資産形成
- 金融商品
- 運用益
- リターン
- 収益
- 複利

代わりに使う表現:

- ボーナス
- ごほうび
- 追加ポイント
- 応援ポイント
- がんばりポイント
- 家庭内ルール
- おこづかいルール
- 親からの承認
- 親からの評価

## 6. 最初のUI改善候補

### 保護者ホーム

- 情報の優先順位を整理する。
- 「未確認の申請」「今月支給したおこづかい」「月次ボーナス」を見やすくする。
- 次にやることカードの文言を現在の機能段階に合わせる。
- 月次ボーナス導線を自然に見せる。

### 月次ボーナス

- 参考値カードをもっと分かりやすくする。
- ボーナス候補を並べて比較しやすくする。
- 「付与する / 付与しない」を誤操作しにくくする。
- 親独自ボーナスは入力しやすくする。
- 付与済み、付与なし、取消済みの履歴を見やすくする。

### こどもホーム

- 現在ポイントを楽しく見せる。
- 申請する導線を目立たせる。
- おこづかい申請はポイントが足りる/足りないが分かるようにする。
