# INCE 3ヵ年事業計画

作成日: 2026-05-29

## 1. 事業概要

INCEは、子どもの学習成果を保護者が確認し、ポイントとして付与し、おこづかい申請までつなげる親子向け学習モチベーションアプリである。

MVPの中心体験は以下とする。

1. 子どもがテスト・成績・学習成果を申請する。
2. 保護者が内容を確認し、承認・差し戻し・却下する。
3. 承認時にポイントを付与する。
4. 子どもがポイントを貯め、おこづかい申請する。
5. 保護者が支給完了する。

課金は保護者1世帯ごとのサブスクリプションとする。

## 2. 市場前提

2024年の国民生活基礎調査では、児童のいる世帯は907.4万世帯、全世帯の16.6%とされている。

INCEの直接対象は、この全体ではなく、以下の条件を満たす家庭とする。

- 子どもの学習習慣づけに課題がある。
- テストや成績を親子で確認している。
- おこづかいを学習成果や家庭内ルールと結びつけることに抵抗が少ない。
- 月500円程度の教育・家庭管理アプリに支払う余地がある。

初期の獲得対象は、児童のいる世帯のうち0.01%から0.15%程度を現実的なレンジとして見る。

## 3. 料金前提

| 項目 | 金額 |
|---|---:|
| 月払い | 500円/月 |
| 年払い | 5,000円/年 |
| 無料トライアル | 14日間 |

収支シミュレーションでは、月払い70%、年払い30%と仮定する。

年間平均単価:

```text
月払い 6,000円 × 70% = 4,200円
年払い 5,000円 × 30% = 1,500円
合計 5,700円/年
```

## 6. 売上成長の考え方

### 1年目: MVP検証と初期課金化

目的:

- 親子の申請・承認・ポイント付与体験が継続利用されるか検証する。
- 30世帯から100世帯の実利用を作り、課金意思を確認する。
- 年末時点で1,000課金世帯を目指す。

主な施策:

- LP公開
- SNS・子育てコミュニティでの検証
- 知人・地域・学校外コミュニティでのテスト導入
- 14日間無料トライアル
- デモ体験導線の整備
- 保護者インタビュー

重点KPI:

- 無料登録率
- 申請作成率
- 保護者承認率
- 2週間後継続率
- 無料から課金転換率

### 2年目: β版拡大と継続率改善

目的:

- 課金世帯数を4,000世帯まで拡大する。
- 解約率を月3.5%程度まで下げる。
- おこづかい申請・ポイント履歴・通知の利用頻度を高める。

主な施策:

- 本番DB・認証・決済導入
- 写真保存・通知・退会・問い合わせ対応の整備
- SEO記事、SNS、紹介施策
- 親子向け利用テンプレートの追加
- 兄弟利用、複数子ども管理の改善
- サポート体制の開始

重点KPI:

- 月次継続率
- 月間申請数/世帯
- 月間承認数/世帯
- 紹介経由登録数
- CAC

### 3年目: 事業化と利益化

目的:

- 年末12,000課金世帯を目指す。
- 通年黒字化する。
- 家庭学習・おこづかい管理カテゴリで認知を作る。

主な施策:

- 広告運用の本格化
- 口コミ・紹介プログラム
- 教育系インフルエンサー、メディア連携
- 年払い比率の向上
- 継続利用レポート、月次振り返り機能
- 家族内ルール管理・ごほうび設計の強化

重点KPI:

- 年払い比率
- LTV
- CAC回収期間
- 月次解約率
- 課金世帯あたり子ども数

## 7. ユニットエコノミクス

| 指標 | 前提 |
|---|---:|
| 年間平均単価 | 5,700円 |
| 月額平均単価 | 475円 |
| 月次解約率 | 3.0〜4.0% |
| 想定平均継続月数 | 25〜33ヵ月 |
| 想定LTV | 約11,875〜15,675円 |
| 目標CAC | 1,500〜3,000円以内 |

月額単価が低いため、広告だけで急拡大するより、紹介・SNS・検索流入・コミュニティ経由を組み合わせ、CACを抑える必要がある。

## 8. 開発計画

### 1年目

- 現在の静的MVPを検証用に公開する。
- 本番用の認証、DB、ストレージ、決済の設計を固める。
- Supabaseまたは同等のBaaSへ移行する。
- Stripe等でサブスクリプション課金を実装する。
- 写真保存、通知、退会、問い合わせを最低限実装する。

### 2年目

- 利用データに基づく改善を進める。
- 月別履歴、振り返り、子ども向けモチベーション表示を強化する。
- 兄弟利用、科目別ルール、成績評価パターンを改善する。
- サポート管理画面を整える。

### 3年目

- 継続率改善のためのレポート機能を追加する。
- 紹介プログラムを実装する。
- 学習習慣化テンプレート、家庭内ごほうび設計を拡張する。
- 法人・塾・地域団体向けの可能性を検証する。

## 9. 必要資金の目安

3年目黒字化を目指す場合、1〜2年目の累計赤字は以下と見込む。

| 見方 | 累計赤字 |
|---|---:|
| 創業者人件費を抑える場合 | 約450万円 |
| 創業者人件費を含める場合 | 約810万円 |

最低限の資金としては、初期開発・広告検証・運用費を含めて500万円前後を確保したい。

外注開発比率が高い場合や、広告で早期拡大する場合は、1,000万円以上の運転資金を見込む。

## 10. 主要リスク

| リスク | 内容 | 対策 |
|---|---|---|
| 課金転換率が低い | 無料では使うが月500円を払わない | 無料期間中に価値を感じる導線を強化する |
| 継続率が低い | 申請が習慣化しない | 通知、履歴、月次振り返りを強化する |
| 子どもの心理的負担 | 勉強とお金の結びつけに抵抗がある | 点数だけでなく努力・その他申請を重視する |
| 保護者の入力負担 | ルール設定や承認が面倒 | デフォルト設定とワンタップ承認を整える |
| CAC高騰 | 月500円では広告回収が難しい | SEO、紹介、コミュニティ経由を重視する |
| 個人情報・写真管理 | 子どもの写真を扱う | 保存期限、削除、セキュリティを本番前に整備する |

## 11. 3年後の到達目標

- 課金世帯数: 12,000世帯
- 年間売上: 約4,275万円
- 営業利益: 約990万円
- 月次解約率: 3.0%以下
- 年払い比率: 30%以上
- 申請・承認・おこづかい申請の中心体験が確立している状態

## 12. 次に検証すべきこと

1. 10〜30世帯にMVPを触ってもらう。
2. 子どもが自分で申請したくなるかを見る。
3. 保護者が承認作業を面倒に感じないかを見る。
4. 月500円を払いたいと言う家庭がどの程度あるか確認する。
5. 継続利用されるポイントルールの形を見つける。

最初の目標は、30世帯中10世帯が「課金してもよい」と答える状態を作ることとする。
