## 動かし方

`index.html` をブラウザで開くか、簡易サーバーで `app` ディレクトリを配信します。

```sh
python3 -m http.server 4173
```

ブラウザで `http://localhost:4173` を開きます。

## 現在の実装メモ

- Phase 1-11では保護者情報、ログイン状態、こども情報、申請情報、おこづかい申請、ポイント履歴、通知、契約状態、運営ログイン状態をブラウザのローカルストレージに仮保存します。
- 運営管理画面は `/admin` です。デモログインは `admin@example.com` / `admin123` です。
- パスワードもプロトタイプ用の仮保存です。本番実装では認証基盤側で安全に扱います。
- 契約状態は仮で `trial` として扱います。
- 決済会社とは未接続です。プラン・支払い設定では契約状態だけをプロトタイプ用に切り替えます。
- 無料トライアル・契約中・猶予期間は利用可能、支払い失敗・解約・期限切れは支払い設定へ誘導します。
- 設定画面からローカル保存データを書き出し・初期化できます。
- 設定画面からデモこども、サンプル申請、ポイント履歴、おこづかい申請を作成できます。
- 設定画面からデモの使い方を開き、確認順とこどもログイン情報を見られます。
- こども追加時に、初期科目と標準ポイントルールの準備データを作成します。
- こども削除は完全削除ではなく、一覧から非表示にする `deleted` 扱いです。
- 科目削除も完全削除ではなく、申請履歴に備えて `deleted` 扱いです。
- 申請写真はプロトタイプではブラウザ内に仮保存します。本番ではStorageへ保存します。
- 写真は承認済み60日、やり直し・却下30日を過ぎると画面表示時に自動削除します。
- キャラクター画像は使用しない方針です。
- npmが利用可能になったら、Next.js / TypeScript / Supabase構成へ移行する前提です。
